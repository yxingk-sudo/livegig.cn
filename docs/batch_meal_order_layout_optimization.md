# 批量报餐页面布局优化说明

## 优化目标
重新组织[/user/batch_meal_order.php](file:///www/wwwroot/livegig.cn/user/batch_meal_order.php)页面的布局结构，按照清晰的报餐流程顺序排列页面元素：
1. 首先显示人员选择区域
2. 其次显示餐类型和套餐信息选择区域
3. 最后显示用餐日期、用餐时间等详细信息填写区域

## 优化内容

### 1. 页面布局重新排序
将原有的页面元素顺序调整为：

1. **人员选择区域**
   - 人员选择方式（按部门选择/按个人选择）
   - 部门选择区域（多选形式）
   - 个人选择区域（包含项目人员列表和已选人员显示）

2. **餐类型和套餐信息选择区域**
   - 餐类型选择（早餐、午餐、晚餐、宵夜）
   - 套餐选择（根据所选餐类型动态加载）

3. **用餐详细信息填写区域**
   - 用餐日期（默认为项目开始日期，限制在项目日期范围内）
   - 用餐时间（默认12:00）
   - 送餐时间（默认11:30）
   - 特殊要求（可选填写）

### 2. 用户体验优化
- 实现了清晰的报餐流程引导：先选人→再选餐→最后填时间
- 保持了原有的所有功能不变
- 界面交互逻辑保持一致

### 3. 技术实现
- 保持了原有的JavaScript交互功能
- 保持了原有的表单验证逻辑
- 保持了原有的AJAX动态加载套餐功能
- 保持了原有的响应式设计

## 修改效果
1. 用户现在可以按照更符合逻辑的顺序进行批量报餐操作
2. 页面布局更加清晰，符合用户的操作习惯
3. 提高了用户操作的效率和准确性
4. 保持了原有功能的完整性和稳定性

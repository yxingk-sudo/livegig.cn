# 报餐日期多选功能 - 使用说明

## 功能概述

将后台的日期范围控制从"开始日期 + 结束日期"的方式改为直接列出所有日期供多选。管理员可以直观地勾选需要显示的日期，前台页面将根据选择动态显示相应日期列。

## 数据库迁移

### 字段变更

**旧字段（已删除）：**
- `meal_start_date` (DATE) - 报餐管理开始日期
- `meal_end_date` (DATE) - 报餐管理结束日期

**新字段：**
- `selected_meal_dates` (TEXT) - 选中的报餐日期（JSON 格式存储）

### 执行迁移

迁移脚本已自动执行：
```bash
php /www/wwwroot/livegig.cn/migrate_selected_dates.php
```

迁移步骤：
1. ✅ 添加 `selected_meal_dates` 字段
2. ✅ 迁移旧数据到新字段（如果有）
3. ✅ 删除旧的 `meal_start_date` 和 `meal_end_date` 字段

## 后台管理功能

### 访问路径
```
/admin/meal_packages.php?project_id=您的项目 ID
```

### 位置
在项目选择后，"餐类型管理"卡片下方，会看到"报餐日期选择"卡片。

### 界面布局

#### 1. 日期列表显示区
- 📅 **按月份分组**：日期按年月分组显示，每个月一个区域
- 📊 **可滚动区域**：最大高度 400px，超出可滚动
- 🏷️ **日期格式**：`YYYY-MM-DD (周 X)`
- ✅ **复选框**：每个日期都有独立的复选框

#### 2. 快捷操作按钮
- ✔️ **全选**：选中所有日期
- ❌ **取消全选**：取消所有选中日期
- 📅 **仅本周**：只选中本周一到周日的日期

#### 3. 状态说明
- ✅ 已勾选的日期：将在前台显示
- ⬜ 未勾选的日期：不会在前台显示
- 📝 数据来源：酒店入住记录

### 使用方法

#### 方式一：手动选择
1. 在日期列表中逐个勾选需要显示的日期
2. 点击"保存日期选择"按钮

#### 方式二：使用快捷按钮
1. **全选**：点击"全选"按钮 → 确认 → 保存
2. **取消全选**：点击"取消全选"按钮 → 确认 → 保存
3. **仅本周**：点击"仅本周"按钮 → 确认 → 保存

#### 方式三：混合选择
1. 先使用快捷按钮进行初步选择
2. 再手动调整个别日期
3. 点击"保存日期选择"按钮

## 前台页面显示

### 访问路径
```
/user/meal_management.php
```

### 过滤逻辑

1. **有选中日期**
   - 只显示选中的日期
   - 即使有酒店记录，未选中的日期也不显示

2. **未选中任何日期**（NULL 或空数组）
   - 显示所有酒店入住记录的日期
   - 保持原有功能不变

### 显示规则

```
后台选中了：2024-01-15, 2024-01-16, 2024-01-20
↓
前台显示：
┌──────────┬──────────┬──────────┐
│01-15(一) │01-16(二) │01-20(六) │
│早 午 晚  │早 午 晚  │早 午 晚  │
└──────────┴──────────┴──────────┘

注意：01-17, 01-18, 01-19 不显示（因为未选中）
```

## 技术实现

### 数据存储格式

使用 JSON 格式存储选中的日期：
```json
["2024-01-15", "2024-01-16", "2024-01-20"]
```

### 后台实现

1. **获取可用日期**
   ```php
   $allDatesQuery = "SELECT DISTINCT check_in_date as date FROM hotel_reports WHERE project_id = :project_id 
                     UNION SELECT DISTINCT check_out_date as date FROM hotel_reports WHERE project_id = :project_id 
                     ORDER BY date";
   ```

2. **解析已选日期**
   ```php
   $selectedDatesArray = json_decode($project_config['selected_meal_dates'], true);
   ```

3. **保存选择**
   ```php
   $selected_meal_dates = !empty($selected_dates) ? json_encode($selected_dates, JSON_UNESCAPED_UNICODE) : null;
   $query = "UPDATE projects SET selected_meal_dates = :selected_meal_dates WHERE id = :project_id";
   ```

### 前台实现

1. **读取配置**
   ```php
   if (!empty($date_config_row['selected_meal_dates'])) {
       $decodedDates = json_decode($date_config_row['selected_meal_dates'], true);
       if (is_array($decodedDates)) {
           $selectedMealDates = $decodedDates;
       }
   }
   ```

2. **过滤日期**
   ```php
   if (!empty($selectedMealDates)) {
       $filteredDates = [];
       foreach ($sortedDates as $date) {
           if (in_array($date, $selectedMealDates)) {
               $filteredDates[$date] = true;
           }
       }
       $sortedDates = array_keys($filteredDates);
   }
   ```

## 优势对比

### 旧方式（开始 + 结束日期）

❌ **缺点：**
- 只能设置连续日期范围
- 无法跳过中间的日期
- 不够灵活

✅ **优点：**
- 设置简单

### 新方式（多选日期）

✅ **优点：**
- 可以选择任意日期组合
- 可以跳过不需要的日期
- 更直观、更灵活
- 支持快捷操作

❌ **缺点：**
- 日期较多时选择较繁琐（已通过快捷按钮解决）

## 常见使用场景

### 场景 1：只在工作日报餐
1. 点击"取消全选"
2. 手动勾选所有周一至周五的日期
3. 跳过周六和周日
4. 保存

### 场景 2：只针对特定活动日期
1. 点击"取消全选"
2. 只勾选活动举办的几天
3. 保存

### 场景 3：默认全开，只排除特殊日期
1. 点击"全选"
2. 取消勾选节假日等特殊日期
3. 保存

### 场景 4：临时调整
1. 点击"仅本周"快速选择当前周
2. 根据实际需要微调
3. 保存

## 注意事项

1. **数据来源**
   - 日期列表来自酒店入住记录
   - 如果没有酒店记录，将显示"暂无酒店入住记录"提示

2. **性能考虑**
   - 日期列表有滚动条，最多显示 400px 高度
   - 按月分组，便于查找

3. **兼容性**
   - 未设置时（NULL）兼容原有功能
   - 自动降级为显示所有日期

4. **数据格式**
   - JSON 格式存储
   - UTF-8 编码，支持中文注释

## 常见问题

### Q1: 为什么看不到可选日期？
**A:** 可能是因为该项目还没有酒店入住记录。请先录入酒店入住数据。

### Q2: 如何恢复为显示所有日期？
**A:** 点击"取消全选"按钮，然后保存即可。或者将 `selected_meal_dates` 设置为 NULL。

### Q3: 选中的日期太多，找不到某个日期怎么办？
**A:** 日期是按月分组的，可以滚动查看。也可以使用浏览器的查找功能（Ctrl+F）。

### Q4: "仅本周"是如何计算的？
**A:** 以当前日期为准，计算本周一到周日的日期范围。

### Q5: 如果今天已经是周日了，"仅本周"会选择哪几天？
**A:** 会选择本周一到今天（周日）的所有日期。

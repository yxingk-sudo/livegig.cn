## 透明度调整详情

### 旧方案（浅淡模式）

背景色透明度过低，视觉效果不够突出：

```css
/* 表行背景 */
background-color: rgba(R, G, B, 0.05);  ← 仅 5% 透明度

/* 行程行背景 */
background-color: rgba(R, G, B, 0.08);  ← 仅 8% 透明度

/* 行程卡片 */
background-color: rgba(R, G, B, 0.06);  ← 仅 6% 透明度
```

**问题**:
- 颜色过浅，与白色背景融合
- 多行表格时难以快速区分车辆
- 整体视觉效果偏淡

### 新方案（深色强调模式）

显著提升背景颜色的透明度，确保对比度：

```css
/* 表行背景 */
background-color: rgba(R, G, B, 0.10);  ← 提升到 10% 透明度 (+100%)

/* 行程行背景 */
background-color: rgba(R, G, B, 0.15);  ← 提升到 15% 透明度 (+87%)

/* 行程卡片 */
background-color: rgba(R, G, B, 0.12);  ← 提升到 12% 透明度 (+100%)
```

**优势**:
- 颜色更加饱和，视觉冲击力强
- 车辆分组明显，易于快速扫视识别
- 行程行与空闲行区分度大幅提升
- 整体表格视觉层级更清晰

## 设计原则

### 1. 颜色饱和度原则

**旧方案**: 使用"高亮色"(Light colors)
- 优点: 柔和，不刺眼
- 缺点: 饱和度低，对比不足

**新方案**: 使用"标准色"(Standard colors)
- 优点: 饱和度高，对比明显
- 缺点: 需要平衡，不过于鲜艳

### 2. 透明度阶梯原则

保持原有的三层阶梯，但整体提升：

```
旧方案          新方案
5%  ← 基础     10% ← 基础 (提升 2 倍)
6%  ← 卡片     12% ← 卡片 (提升 2 倍)
8%  ← 行程     15% ← 行程 (提升 1.87 倍)
```

**目的**: 保持层级关系，同时提升整体对比度

### 3. 颜色可辨性原则

选择的 6 种新颜色需要满足：
- 相邻颜色对比度高 (不容易混淆)
- 符合 Web 标准色系 (专业、广为认知)
- 支持色盲用户 (不仅依赖颜色区分，还有亮度差异)
